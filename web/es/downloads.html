Paquetes debian / ubuntu más recientes

Utilice el siguiente repositorio APT para instalar MyQtt y sus dependencias

NOTA: no use estas fuentes si está utilizando Core-Admin a menos que sepa qué está haciendo.


  # repositorio ubuntu publico de aspl (PRECISE)
  deb http://www.aspl.es/ubuntu/public precise/

  # repositorio debian publico de aspl (WHEEZY)
  deb http://www.aspl.es/debian/public wheezy/

  # repositorio debian publico de aspl (SQUEEZE)
  deb http://www.aspl.es/debian/public squeeze/

  # repositorio debian publico de aspl (LENNY)
  deb http://www.aspl.es/debian/public lenny/

Luego ejecute apt-cache search myqtt para buscar los paquetes para MyQtt.

Pquetes para CentOS

Utilice los siguientes repositorios para obtener acceso a los paquetes para CentOS:

NOTA: no use estas fuentes si está utilizando Core-Admin a menos que sepa qué está haciendo.


  # /etc/yum.repos.d/ASPL-Base.repo
  # Aspl-Public repo 
  #
  # ASPL packages for libaxl, vortex, nopoll, myqtt, ext-dns, valvula
  # and other open source public projects
  #
  
  [aspl-public]
  name=Aspl-Public-$releasever - Base
  baseurl=http://www.aspl.es/centos/public/$releasever/$basearch/
  gpgcheck=0
 
  # Aspl-Private repo 
  #
  # ASPL packages for commercial projects like core-admin
  #
  [aspl-private]
  name=Aspl-Private-$releasever - Base
  baseurl=http://www.aspl.es/centos/private/$releasever/$basearch/
  gpgcheck=0

Luego ejecute yum search myqtt para buscar paquetes myqtt.
